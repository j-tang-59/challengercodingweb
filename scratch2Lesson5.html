Lesson 5: Flappy Bird - Foundational Components

Overview:

In this lesson, we will focus on teaching foundational components needed to create a Flappy Bird-style game. You will learn key elements like gravity, flapping, scrolling backdrops, and a scoring mechanism. Each exercise builds toward understanding the mechanics, but you won’t be making the entire game just yet!

Exercise 1: Flapping - Gravity and Costume Change

In this exercise, you will learn how to implement gravity, where the bird is pulled downward constantly, and a "flap" mechanism where the bird rises when the player presses a key. Additionally, we'll change the bird's costume during the flap to simulate wing movement.

Step 1: Implementing Gravity

The bird should always be pulled downward by gravity. We’ll create a variable to control this downward movement. What Scratch feature can we use to represent a force pulling the bird down?

Steps for Gravity:
  1. Create a variable called Gravity to control how fast the bird falls. Set it to -1 initially.
  2. Use a forever loop and inside it, use change y by [Gravity] to continuously apply downward force to the bird’s y-position.
  3. Increase the gravity value (e.g., change Gravity by -0.1) each time through the loop to simulate faster falling over time.
  4. To limit the fall speed, use a condition like if Gravity < -5, set Gravity to -5.

Step 2: Flapping to Rise

When the player presses the spacebar, the bird should rise. What block can we use to detect a key press and change the bird’s motion?

Steps for Flapping:
  1. Use the when [space] key pressed block to detect when the player presses the spacebar.
  2. When the spacebar is pressed, set Gravity to a positive value (e.g., 10) to make the bird rise temporarily.
  3. After flapping, gravity should begin pulling the bird down again by gradually reducing the Gravity value back to a negative number.

Step 3: Changing Costume for Wing Movement

We want to visually show the bird flapping its wings when the player presses the spacebar. How can we make this happen?

Steps for Changing Costume:
  1. Draw or import a second costume for the bird with the wings raised.
  2. Use the switch costume to [flap] block when the spacebar is pressed to show the bird flapping its wings.
  3. After a short delay (e.g., 0.2 seconds), switch back to the normal costume using switch costume to [normal].

Exercise 3: Implementing a Scrolling Backdrop with Clones

In this exercise, we will create a scrolling backdrop using **clones**. Cloning is a powerful feature in Scratch that allows us to create multiple copies of the same sprite, making it look like the world is endlessly moving without needing to reset the backdrop.

Step 1: Introduction to Cloning

Instead of just moving a single backdrop left and resetting it when it leaves the screen, we can create multiple clones of the backdrop to continuously scroll across the stage. Each clone will follow the same behavior, giving the illusion of an endless background.

Steps for Cloning the Backdrop:
  1. Create a Backdrop Sprite: Draw or import a sprite that represents a part of the scrolling background, like clouds, mountains, or buildings.

  2. Set Up Cloning: In the when green flag clicked block, create a clone of the backdrop every few seconds using the create clone of [myself] block.
    Example: Use repeat or forever blocks to create a clone every 2 seconds to keep the scrolling effect continuous.

  3. Move the Clones: Inside the when I start as a clone block, use the change x by [-3] block to make each clone move left across the screen.

  4. Delete the Clones When Offscreen: To ensure the clones don’t keep moving off-screen forever, add a condition inside the clone loop. Example: if x < -240 (when the sprite moves off the left edge), use delete this clone.

Step 2: Testing the Scrolling Backdrop

Now that we’ve set up cloning and movement, it’s time to test your scrolling backdrop.

  • Make sure new clones of the backdrop appear at regular intervals.
  • Check that each clone moves left and gets deleted when it goes off the screen.
  • Adjust the timing between clone creation if needed to maintain a seamless background.

Exercise 4: Implementing a Scoring Mechanism

In Flappy Bird, the player scores points each time they pass an obstacle. How can we track and update the player’s score in Scratch?

Steps for Scoring:
  1. Create a variable called Score to track the player's points.
  2. Every time the bird successfully passes an obstacle, use change Score by 1 to increase the player's score.
  3. Display the score on the screen by checking the "show" option next to the Score variable.
