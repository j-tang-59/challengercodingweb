Lesson 5 Guide

Vocabulary:

  • Conditional: A conditional is a statement that performs different actions based on whether a certain condition is true or false.
  • Boolean: A boolean is a data type that represents one of two values: true or false. Booleans are often used in conditionals to make decisions in code.
  • While Loop: A while loop is a type of loop that keeps repeating a set of instructions as long as a certain condition is true. This is a loop and a conditional.
  • If Statement: An if statement is a type of conditional that executes a set of instructions only if a certain condition is true.
  • If Else Statement: An if-else statement is a type of conditional that executes one block of code if a condition is true and another block of code if the condition is false.

Blocks Used/Introduced Today:

  • The While Loop. Use this block to create the conditional to where the while loop will execute.

  • If Conditional. If the conditional is met, the code inside will execute.

  • If Else Conditional. Allows you to program the conditional if the conditional is not met.

  • This block lets your conditional be checked. The computer will check if the key you selected has been pressed or not.

  • Touching Color. This block checks if the sprite is touching a specified color. Often used in conditionals to trigger an action when the sprite interacts with a specific color on the stage.

  • Key Pressed. This block checks if a specific key on the keyboard is pressed. It is commonly used in conditionals to trigger actions based on user input.

  • Mouse Down. This block checks if the mouse button is being pressed. It can be used to create interactive elements where the sprite reacts to mouse clicks.

Exercise 1: Implementing Conditionals

A conditional in coding is a statement that allows a program to execute a certain piece of code only if a specific condition is met.

Conditionals are very versatile and important. It allows programs to handle multiple scenarios and basically allows the computer to make decisions!

Examples include:

  • Computer log in with the correct password
  • If temperature is too hot, stay inside

    • A conditional check is when the computer "checks" whether a certain condition, like pressing a key, is met. Think of it when someone asks you a yes no question. This is how conditionals are executed.


    If Conditional: Use the "If" block to make a decision in your code. The sprite will only execute the action if the condition inside the block is true.


    If Else Conditional: Use the "If Else" block when you need the sprite to perform one action if the condition is true, and a different action if it is false.


    While Loop: Although the while loop is technically a loop, it is also a conditional. Use the "While" block when you want to repeat an action until a certain condition is no longer true.

    Exercise 2: Implementing Common Conditionals in Scratch

    In this exercise, you'll learn how to use common conditional blocks in Scratch to make your sprite respond to different events and conditions.


    Understanding Conditionals:

    A conditional is like a decision-maker in your program. It helps the computer decide what to do next based on whether a certain condition is true or false.


    For example, if you want your sprite to move only when the space key is pressed, you would use a conditional to check if the space key is pressed before moving the sprite.


    Step 1: Using the "Key Pressed" Conditional

    Let's start by making your sprite move when a key is pressed:

    1. Drag the "When Green Flag Clicked" block to the workspace.

    2. Go to the "Control" category and drag the "If" block into the workspace.

    3. In the "Sensing" category, find the "Key [space] pressed?" block and drag it into the hexagon space in the "If" block.

    4. Inside the "If" block, add a "Move 10 steps" block from the "Motion" category.


    Now, when you press the space key, your sprite will move 10 steps!


    Step 2: Using the "Touching Color" Conditional

    Next, let's make your sprite change direction when it touches a specific color:

    1. Below your previous "If" block, add another "If" block.

    2. Go to the "Sensing" category, and find the "Touching color [ ] ?" block.

    3. Drag this block into the hexagon space in the new "If" block. Click the color box and choose a color that your sprite will interact with.

    4. Inside this "If" block, add a "Turn 15 degrees" block from the "Motion" category.


    Now, whenever your sprite touches the specified color, it will turn 15 degrees!


    Step 3: Combining Conditionals with Loops

    Finally, let's combine what we've learned with a loop:

    1. Wrap both of your "If" blocks inside a "Forever" loop from the "Control" category.

    2. This way, the conditions will be continuously checked, and your sprite will keep responding to key presses and touching colors as long as the program runs.


    Your sprite is now interactive! It moves when you press the space key and changes direction when it touches a certain color, all thanks to conditionals and loops.

    Maze Game Enhancement: Adding Arrow Keys and "Death" by Touching Walls

    In this tutorial, you’ll enhance your maze game by adding controls with the arrow keys and a feature where the sprite "dies" (or restarts) if it touches the walls of the maze.


    Step 1: Setting Up the Arrow Keys

    Let's start by making your sprite move with the arrow keys:


    1. Drag the "When Green Flag Clicked" block to the workspace.

    2. Go to the "Control" category and drag a "Forever" loop into the workspace. This will keep the sprite moving as long as the program runs.


    3. Inside the "Forever" loop, add four "If" blocks from the "Control" category.

    4. Go to the "Sensing" category, and in each "If" block, add a "Key [arrow] pressed?" block. Set the first block to "Key Up arrow pressed?", the second to "Key Down arrow pressed?", the third to "Key Right arrow pressed?", and the fourth to "Key Left arrow pressed?".


    5. Inside each "If" block, add a "Move 10 steps" block from the "Motion" category. For the "Up" and "Down" arrows, also add a "Point in direction" block and set it to 0 (Up) and 180 (Down) degrees, respectively. Set the "Right" arrow to 90 degrees and the "Left" arrow to -90 degrees.


    Now, your sprite should move in the direction of the arrow key you press!


    Step 2: Adding Walls to the Maze

    Next, ensure your maze has walls that the sprite cannot pass through:


    1. Draw or update your maze backdrop to include clear walls.

    2. Go to the "Costumes" tab for your sprite, and draw your sprite (or use an existing one).


    Make sure the walls and sprite are distinct colors for easy detection.


    Step 3: Adding the "Death" Functionality

    Now, let’s make your sprite "die" (or restart) when it touches the walls:


    1. Inside the "Forever" loop, under your arrow key conditions, add another "If" block.

    2. Go to the "Sensing" category, and add a "Touching color [ ]?" block into the "If" block. Click the color box and choose the color of your maze walls.


    3. Inside the "If" block, add a "Go to [start position]" block from the "Motion" category. Set the X and Y coordinates to the starting position of your maze.

    4. Optionally, you can add a "Play sound [sound]" block to play a sound when the sprite "dies."


    Now, if the sprite touches the walls, it will restart at the beginning of the maze!


    Step 4: Testing Your Maze Game

    Test your game by pressing the green flag and using the arrow keys to navigate your sprite through the maze. Make sure the sprite correctly "dies" and restarts if it touches the walls.


    Congratulations! You’ve now enhanced your maze game with arrow key controls and a "death" mechanic when touching the walls!
