Lesson 1: Calculator

Overview:

In this lesson, you will create a simple calculator in Scratch. By the end of this project, you’ll have a functioning calculator that can perform basic math operations (addition, subtraction, multiplication, and division).

Exercise 1: Ask Function & Conditional Logic

Section 1: The Ask and Wait Block

In Scratch, the ask [ ] and wait block allows us to ask the user a question and wait for their response. It’s a great way to get input from the user, such as asking for a number or an operation they want to perform.

This block belongs to the "Sensing" category. When you drag the ask [ ] and wait block into your workspace, you’ll notice a white circle where you type your question. After the user answers, Scratch stores their response in the answer block, which you can use later in the program.

For example, if you ask, "What's your name?" the user’s response will be stored in the answer block, and you can use it in your program.

Section 2: Transition into the Calculator

Now that you’ve learned about the ask [ ] and wait block, let's think about how we can use it in our calculator project. You need to ask the user a question. What would be the most important question to ask in a calculator?

Question: "What operation would you like to perform? (add, subtract, multiply, divide)"

Now that you know the question, what should you do next with the user's response? Think about how you would store the answer.

The user's response is automatically stored in the answer block, which you can use later to control the flow of the program.

Section 3: Creating the Conditional

Now, how do you think we can decide which operation the user chose? What block could help us check if the user typed "add", "subtract", "multiply", or "divide"?

We can use an if [ ] then block from the "Control" tab to create a condition that checks what the user typed.

Next, how would you check if the user's input matches "add"? What operator could help you with this?

Use the = block from the "Operators" tab to check if the answer equals "add".

Once you've done that, place the answer block on the left side of the equals operator, and type "add" on the right side. If the user's input matches, you'll be ready to perform the addition operation.

Section 4: Adding the Broadcast Block

Now, how can we notify other parts of the program that the user has selected "add"? How can we broadcast this choice across the program?

We can use the broadcast [message] block from the "Events" tab to send a signal to the rest of the program.

Once you’ve added the broadcast block, create a new message and name it "addition". This will notify the rest of the program to perform the addition operation.

In the next exercise, we’ll guide you through implementing the actual math operations. For now, test your program to make sure it asks for the operation and sets up the correct broadcast signal.

Click the green flag to test your program!

Exercise 2: Coding the Subtraction Operation

In this exercise, you will implement the subtraction operation for the calculator. The conditional and broadcast setup has already been completed in Exercise 1, so now we will focus solely on the subtraction logic.

Step-by-Step Instructions:

First, how do you think we can ask for the two numbers that we need to subtract? What blocks should we use to store these inputs?

Use the ask [ ] and wait block to ask for the first number, and store the response in a variable called number1.

Then, repeat the same step for the second number and store it in a variable called number2.

Now that you have the two numbers stored, how can we perform the subtraction? What block can we use to get the sprite to say the result?

Instead of using a set block, you can use the say [ ] block to show the result. To do this, go to the "Operators" tab and find the subtraction block (-).

Place the number1 and number2 variables inside the subtraction operator, and then place that inside the say [ ] block.

The sprite will now say the result of number1 - number2 directly!
